# ⏰ 如何设置执行时间

## 🎯 方法 1: 使用可视化 UI（推荐）

### 步骤：

1. **进入 Actions 页面**
   - 打开你的 GitHub 仓库
   - 点击 **Actions** 标签页

2. **选择「更新定时执行时间」工作流**
   - 在左侧工作流列表中找到 **「更新定时执行时间」**
   - 点击进入

3. **设置执行时间**
   - 点击右上角的 **Run workflow** 下拉按钮
   - 从下拉菜单中选择：
     - **第一次执行时间**：例如 `09:00` 或 `09:30`（北京时间）
     - **第二次执行时间**：例如 `21:00` 或 `21:30`（北京时间）
   - 支持整点和半点时间（如 09:00, 09:30, 10:00, 10:30...）
   - 共 36 个时间点可选（06:00 - 23:30，每半小时一个）
   - 点击绿色的 **Run workflow** 按钮

4. **等待更新完成**
   - 工作流会自动运行（约 10-20 秒）
   - 完成后会显示 ✅ 成功标记
   - 新的执行时间立即生效

## 🛠️ 方法 2: 手动编辑 workflow 文件

如果你更喜欢手动编辑：

1. **编辑文件**
   - 打开 `.github/workflows/refresh.yml`
   - 找到第 13 行

2. **修改 cron 表达式**
   ```yaml
   schedule:
     - cron: '0 1,13 * * *'  # 修改这一行
   ```

3. **时间对照表**

| 北京时间 | UTC 时间 | cron 表达式 |
|---------|---------|------------|
| 06:00 | 22:00 (前一天) | `0 22 * * *` |
| 07:00 | 23:00 (前一天) | `0 23 * * *` |
| 08:00 | 00:00 | `0 0 * * *` |
| 09:00 | 01:00 | `0 1 * * *` |
| 10:00 | 02:00 | `0 2 * * *` |
| 11:00 | 03:00 | `0 3 * * *` |
| 12:00 | 04:00 | `0 4 * * *` |
| 13:00 | 05:00 | `0 5 * * *` |
| 14:00 | 06:00 | `0 6 * * *` |
| 15:00 | 07:00 | `0 7 * * *` |
| 16:00 | 08:00 | `0 8 * * *` |
| 17:00 | 09:00 | `0 9 * * *` |
| 18:00 | 10:00 | `0 10 * * *` |
| 19:00 | 11:00 | `0 11 * * *` |
| 20:00 | 12:00 | `0 12 * * *` |
| 21:00 | 13:00 | `0 13 * * *` |
| 22:00 | 14:00 | `0 14 * * *` |
| 23:00 | 15:00 | `0 15 * * *` |

4. **示例**

想要在北京时间 **08:00** 和 **20:00** 执行：

```yaml
schedule:
  - cron: '0 0,12 * * *'  # UTC 00:00 和 12:00
```

## ⚠️ 注意事项

1. **间隔时间**
   - 建议两次执行间隔至少 8 小时
   - 京东规定刷新后需等待 8 小时才能再次刷新
   - 间隔太短会导致第二次执行处于冷却期

2. **时区说明**
   - 所有时间输入使用**北京时间**（UTC+8）
   - 系统会自动转换为 UTC 时间
   - 无需手动计算

3. **生效时间**
   - 使用 UI 更新：立即生效
   - 手动编辑文件：提交后立即生效
   - 下次执行按新时间计算

4. **查看下次执行时间**
   - 进入 Actions → 京东校园招聘活跃度刷新
   - 查看工作流详情页面
   - GitHub 会显示下次计划执行时间

## 💡 常见问题

### Q: 我修改了时间，但没有生效？

A: 检查：
1. 确认修改已提交到仓库
2. 查看 Actions 是否启用
3. 查看 workflow 文件是否正确更新

### Q: 可以设置每小时执行一次吗？

A: 不建议，原因：
- 京东有 8 小时冷却期
- 频繁请求可能导致封号
- 建议间隔至少 8-12 小时

### Q: 如何临时禁用定时执行？

A: 两种方法：
1. 编辑 workflow 文件，注释掉 `schedule` 部分
2. 在 Actions 页面禁用该 workflow

### Q: 如何恢复默认时间？

A: 使用 UI 设置为：
- 第一次：09:00
- 第二次：21:00
